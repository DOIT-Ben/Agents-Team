# 文档地图

本目录把公开 Beta 指南和工程历史记录分开，让新用户先看试用路径，让维护者再追溯设计、计划和审计证据。

## 先看这里

| 文档 | 适用场景 |
| --- | --- |
| [Beta quickstart](beta-quickstart.md) | 你想用大约 10 分钟试用 Agents-Team。 |
| [Usage guide](usage.md) | 你需要初始化、执行、QA 和管理命令细节。 |
| [Feedback guide](feedback.md) | 你想反馈安装、初始化、执行、QA 或文档问题。 |
| [Feedback privacy](privacy-feedback.md) | 你需要确认哪些信息可以反馈、哪些不该提交。 |
| [Validation lessons](validation-lessons.md) | 你想了解真实仓库验证中暴露出的机制问题。 |

## 工程历史记录

`superpowers/` 目录保存构建机制时产生的设计说明、实施计划和审计记录。这些文件作为公开工程历史和机制证据保留，不是新用户的第一阅读路径。

| 目录 | 内容 |
| --- | --- |
| `superpowers/specs/` | 设计规范和协议决策。 |
| `superpowers/plans/` | 历史实施计划。 |
| `superpowers/audits/` | 审查和验证记录。 |

## 反馈原则

Agents-Team 不做静默遥测。反馈应由用户审阅、脱敏，并通过 GitHub Issue 或其他用户明确认可的渠道主动提交。
