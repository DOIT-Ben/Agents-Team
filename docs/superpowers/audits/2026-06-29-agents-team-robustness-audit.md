# Agents-Team 鲁棒性审计与修复记录

日期：2026-06-29

## 审计结论

Agents-Team 的方向是正确的：通用机制放在 Plugin，项目级规则、Issue/PR 模板、验证脚本和 CI 承担落地。但首版仍有几个会影响即插即用的薄弱点。

## 发现的问题

### P1：分发包不能完全自证

解包 `dist/agents-team-0.1.0.zip` 后运行插件测试会失败。原因是包内包含插件测试，但没有包含仓库根目录的 `tools/build_distribution.py`。这会让使用者无法从分发包复现验证过程。

### P1：初始化覆盖策略过宽

旧逻辑只要目标文件包含 `Generated by agents-team` 标记，就允许覆盖。项目团队一旦手工调整生成模板，再次初始化或升级时就可能丢失本地修改。

### P1：PR 证据契约不够硬

PR 模板要求填写证据，但没有自包含检查器。Codex 仍可能保留 `Closes #`、`待验收` 或空证据，然后把“模板存在”误判为“流程合格”。

## 本轮修复

- 分发构建脚本将 `tools/` 纳入包内，让压缩包可以自证。
- 分发验证脚本在解包后重新运行插件测试，再执行一次临时项目初始化和自校验。
- 初始化覆盖策略改为基于 `managedFiles` 哈希判断；本地漂移和非受控文件会阻止覆盖。
- 新增 `validate_pr_contract.py`，用于检查 PR 正文是否包含关联 Issue、Goal 对照、必须完成证据、验收门禁结果、任务边界检查、QA 结论、剩余风险和回滚方式。
- PR 模板升级为更明确的证据结构。

## 本地验证

```bash
cd plugins/agents-team
python3 -m unittest discover -s tests -v
# 47 tests passed

cd ../..
python3 tools/build_distribution.py --output dist/agents-team-0.1.0.zip
python3 tools/verify_distribution.py dist/agents-team-0.1.0.zip
# status: valid
```
