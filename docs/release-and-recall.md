# 发布、回滚与召回

Agents-Team 使用 `internal`、`beta`、`stable` 三个通道。Beta 通过 GitHub Pre-release 和固定标签分发；稳定通道在全部退出门槛通过前保持关闭。

发布前必须完成完整测试、分发包验证、SHA-256、最小 SBOM、兼容矩阵、已知问题和回滚演练。版本支持当前 Beta 与上一可用 Beta；升级失败时退回上一固定标签。

## 问题版本召回

1. 将 Release 标记为 **Withdrawn**，说明影响范围和停止使用时间。
2. 从推荐安装说明移除受影响标签，稳定通道不得指向问题版本。
3. 发布替代版本或上一已知安全版本的固定回滚命令。
4. 通过原邀请渠道通知测试者，并记录“已升级、已回滚、不再使用、无法联系”。
5. 对仍在运行的问题版本进行二次提醒，复盘召回覆盖率。

Agents-Team **不支持远程强制卸载**。已经安装的插件只能由用户升级、回滚或卸载。

## 退出与删除

先运行 `manage_project.py remove REPO` 查看项目适配器移除范围，再由用户确认 `--apply`。运行日志通过 `delete-logs` 先预览再删除；已经导出的反馈包是普通本地文件，由用户决定保留或删除。
