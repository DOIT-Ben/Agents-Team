# Goal Issue Contract

Required order:

1. **Goal**: one observable end state; no “optimize”, “try”, or “improve” without measurable behavior.
2. **必须完成**: exhaustive checklist; any missing item blocks QA.
3. **验收门禁**: exact automated commands, user behavior, and required evidence. If coverage does not exist, add it.
4. **任务边界**: explicit prohibited files, refactors, data, services, and scope expansion.
5. **风险等级**: L1/L2/L3 with reason.
6. **依赖与阻塞条件**: external prerequisites; ordinary bugs are not blockers.

Tests passing are necessary, not sufficient. Goal, checklist, tests, behavior, and boundaries must all pass.
