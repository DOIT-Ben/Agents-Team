# Core Protocol

The mechanism serves one project owner working through one primary Codex with optional workers. It does not simulate permanent engineering seats.

- One primary Codex owns clarification, risk classification, coordination, integration, and user reporting.
- Workers are temporary and receive non-overlapping scopes.
- GitHub Issue is the source of commitment; PR is the source of implementation evidence; independent QA owns the acceptance verdict.
- Static repository documents never store current task status.
- The user decides product direction, irreversible or costly actions, L3 design, and production release. Ordinary implementation and rework remain autonomous.
- Evidence outranks narrative. Code complete, CI green, QA PASS, production deployed, and release complete are distinct states.
